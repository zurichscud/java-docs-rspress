# 微信网页授权

在微信浏览器（内置浏览器/微信客户端）中获取用户 `openid` 用于支付，核心思路是通过 **微信网页授权（OAuth2.0）** 机制。

## 前置条件

### 1.**已认证服务号**：

- 已认证服务号
- 已认证的政府、事业单位、媒体类公众号，也拥有该权限。

::: warning

其他类型账号均不支持微信网页授权登录功能，例如订阅号

:::

### 2.开通微信支付

在 [微信支付商户平台](https://pay.weixin.qq.com/) 完成签约，并绑定公众号

### 3.配置授权域名

微信公众平台 → 设置与开发 → 公众号设置 → 功能设置 → **网页授权域名**

## 授权流程

### 1.用户打开 H5 页面

前端先判断 URL 有没有 `code`

```
https://h5.xxx.com/pay?courseId=1
```

### 2.没有 code → 跳微信授权

前端跳转到：

```
https://open.weixin.qq.com/connect/oauth2/authorize?
appid=公众号appid
&redirect_uri=https%3A%2F%2Fh5.xxx.com%2Fpay
&response_type=code
&scope=snsapi_base
&state=123
#wechat_redirect
```

参数解释：

- `appid`：公众号 AppID
- `redirect_uri`：授权成功回调地址，需要和微信公众平台的网页授权域名的相同
- `scope=snsapi_base`：静默授权（只拿 openId）
- `state`：自定义参数

用户在微信内打开时会微信自动授权，然后重定向回：

```
https://h5.xxx.com/pay?code=abc123&state=123
```

### 3.前端拿到 code

code 不是 openId，只是一个临时票据。

```js
const code = new URLSearchParams(location.search).get("code")
```

### 4.前端把 code 发给后端

```js
await api.getOpenId(code)
```

### 5. 后端调微信接口换 openId

后端请求微信：

```http
https://api.weixin.qq.com/sns/oauth2/access_token?
appid=你的appid
&secret=你的secret
&code=abc123
&grant_type=authorization_code
```

微信返回：

```json
{
  "access_token": "...",
  "expires_in": 7200,
  "refresh_token": "...",
  "openid": "oAbc123xxxx",
  "scope": "snsapi_base"
}
```

后端拿到：

```
openid = oAbc123xxxx
```

## Q&A

### redirect_uri域名与后台配置不一致，错误码:10003

需要使用已认证的服务号，并且要和在微信公众平台「设置与开发 - 账号设置 - 功能设置 - 网页授权域名」的配置一致
