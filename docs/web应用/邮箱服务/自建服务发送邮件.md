# 自建服务发送邮件

在互联网早期，任何人都可以随便架设一台服务器到处发邮件，导致垃圾邮件和诈骗邮件满天飞。为了解决这个问题，现在的全球邮件接收方（如腾讯、微软、谷歌）建立了一套极其严格的**信任与反垃圾机制**。

## 互联网反垃圾邮件信任机制

如果你用自己的服务器直接发信，接收方服务器会进行以下“灵之拷问”：

- **你是谁？（IP 信用）**：你的云服务器 IP 通常是租来的。如果这个 IP 曾经被别人用来发过垃圾邮件，它早就进了全球黑名单。
- **你合法吗？（域名验证）**：大厂邮箱都配置了严格的 **SPF、DKIM、DMARC** 协议（可以理解为域名的“数字身份证”）。如果没有这些配置，接收方会认为你在冒充他人。

::: tip 为什么要用账号

 当你用 QQ/163 邮箱账号发送时，你的 Spring Boot 只是个“搬运工”，真正把邮件送出去的是腾讯或网易的服务器。

接收方一看是“腾讯的服务器”发来的，且通过了账号密码验证，就会直接信任并放行。

:::

自建邮件服务器把邮件成功投递到用户收件箱的难度，远比写代码难得多。

- **自建服务器**：由于缺乏长期积累的 IP 信誉度，你发出去的邮件大概率会**直接进用户的垃圾箱**，甚至被对方服务器的防火墙**直接拒收（Drop）**，连错误提示都没有。
- **使用大厂账号**：网易、腾讯、谷歌的邮件服务器每天在和全球其他服务器进行百亿级的通信，它们拥有**全网最高级别的白名单和信誉度**。借用它们的账号，能确保你的验证码或通知在几秒钟内稳妥地送达用户的收件箱。
