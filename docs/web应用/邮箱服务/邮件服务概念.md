# 邮件服务

## 邮件系统协议

**SMTP (Simple Mail Transfer Protocol - 简单邮件传输协议)**

- **职责**：**负责发送邮件**。
- **角色**：它是你的 Spring Boot 应用与邮件服务器之间，以及邮件服务器彼此之间通信的语言。你的后端代码就是通过 SMTP 协议把邮件“推”给邮件服务商的。

**POP3 (Post Office Protocol 3 - 邮局协议第三版)**

- **职责**：**负责接收邮件**（离线模式）。
- **特点**：客户端把邮件下载到本地后，通常会从服务器上删除。不支持多端同步。

**IMAP (Internet Message Access Protocol - 因特网消息访问协议)**

- **职责**：**负责接收邮件**（在线同步模式）。
- **特点**：客户端与服务器双向同步。你在手机上读了邮件，网页端也会显示已读。

## 授权码

你需要登录邮箱网页端，在“设置 -> 账户”中开启 SMTP 服务。

系统会为你生成一串独立字符串，这就是**授权码**。

在 Spring Boot 的 `spring.mail.password` 中，填入这个授权码作为密码。



##  邮件格式

Spring Boot 支持发送两种主要类型的邮件：

**SimpleMailMessage（简单文本邮件）**：纯文本，不支持样式。适合发一句话通知、纯数字验证码。

**MimeMessage（多用途互联网邮件扩展）**：

- 支持 **HTML 标签**（可以把邮件排版得非常漂亮）。
- 支持 **内嵌图片/静态资源**。
- 支持 **添加附件**（如 PDF、Excel 报表）。



## spring-boot-starter-mail

Spring 官方提供了 `spring-boot-starter-mail` 起步依赖，它的底层封装了 JavaMail（现在叫 Jakarta Mail）。它为我们提供了一个核心抽象接口：**`JavaMailSender`**。

你只需要在 `pom.xml` 中引入依赖，并在配置文件中填入 SMTP 服务器地址、端口和授权码，Spring Boot 就会自动帮你注入 `JavaMailSender` 实例，你直接调用 `send()` 方法即可。
