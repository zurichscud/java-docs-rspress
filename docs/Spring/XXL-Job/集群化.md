# 集群化







## 分片广播

当一个任务数据量极大（比如要处理 1000 万条数据），一台机器跑太慢，想让所有执行器一起跑，该怎么办？

- **原理**：选择“分片广播”路由策略时，调度中心会向**所有在线的执行器**都发送一个调度请求，但每次请求中会带上两个参数：`index`（当前执行器序号，从 0 开始）和 `total`（总执行器数）。
- **代码逻辑**：你在写业务代码时，可以通过 `XxlJobHelper.getShardIndex()` 和 `getShardTotal()` 拿到这两个值。然后利用 `id % total == index` 这样的 SQL 或过滤条件，让每台机器只捞取属于自己的那一份数据，实现真正的分布式并行处理。
