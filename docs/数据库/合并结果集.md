# 合并结果集

`UNION ALL` 是 SQL 中用于**合并多个查询结果集**的操作符，它会把多个 `SELECT` 的结果直接拼接起来，并且**保留重复数据**。

## 基本语法

```sql
SELECT column1, column2
FROM table1

UNION ALL

SELECT column1, column2
FROM table2;
```

例如：

表 `student_a`

| id   | name |
| ---- | ---- |
| 1    | 张三 |
| 2    | 李四 |

表 `student_b`

| id   | name |
| ---- | ---- |
| 1    | 张三 |
| 4    | 赵六 |

执行：

```
SELECT id, name FROM student_a
UNION ALL
SELECT id, name FROM student_b;
```

结果：

| id   | name |
| ---- | ---- |
| 1    | 张三 |
| 2    | 李四 |
|      | 张三 |
| 4    | 赵六 |

## 自动去重

使用`UNION`可以自动去重

```sql
SELECT name FROM student_a
UNION
SELECT name FROM student_b;
```

`UNION` 去重是根据 **所有 SELECT 返回的列组合值** 进行去重的，而不是根据某一列。

```sql
SELECT id, name
FROM user_a

UNION

SELECT id, name
FROM user_b;
```

数据：

user_a：

| id   | name |
| ---- | ---- |
| 1    | 张三 |
| 2    | 李四 |

user_b：

| id   | name |
| ---- | ---- |
| 3    | 张三 |
| 1    | 张三 |

结果：

| id   | name |
| ---- | ---- |
| 1    | 张三 |
| 2    | 李四 |
| 3    | 张三 |

解释：`(1,张三)` 和 `(1,张三)` 重复，所以去掉一个。

## 过滤

```sql
SELECT id, name
FROM user_a
WHERE name = '张三'

UNION ALL

SELECT id, name
FROM user_b
WHERE name = '张三';
```

## 排序

`UNION` / `UNION ALL` 的排序规则是：**只能在整个合并结果集最后使用 `ORDER BY` 排序**。

```
user_a结果
     +
user_b结果
     ↓
合并后的完整结果
     ↓
ORDER BY
```

```sql
SELECT id, name
FROM user_a

UNION ALL

SELECT id, name
FROM user_b

ORDER BY id DESC;
```

| id   | name |
| ---- | ---- |
| 4    | 赵六 |
| 3    | 王五 |
| 2    | 李四 |
| 1    | 张三 |



## 分页

如果把分页写在 `UNION ALL` 外层，那么分页针对的是合并后的完整数据集。

```sql
SELECT *
FROM (
    SELECT id,name,create_time
    FROM user_a

    UNION ALL

    SELECT id,name,create_time
    FROM user_b
) t
ORDER BY t.create_time DESC
LIMIT 0,20;
```
