# 购物

## product_category

商品分类表

```sql
CREATE TABLE product_category (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '分类ID',

    parent_id BIGINT NOT NULL DEFAULT 0 COMMENT '父分类ID',

    ancestors VARCHAR(500) NOT NULL DEFAULT '' COMMENT '祖级列表，例如 0,1,2',

    level TINYINT NOT NULL DEFAULT 1 COMMENT '层级(1-一级分类 2-二级分类 3-三级分类)',

    category_name VARCHAR(100) NOT NULL COMMENT '分类名称',

    sort_order INT NOT NULL DEFAULT 0 COMMENT '显示排序',

    status TINYINT NOT NULL DEFAULT 1 COMMENT '状态(1-启用 0-禁用)',

    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    KEY idx_parent_id(parent_id),
    KEY idx_level(level)
) COMMENT='商品分类表';
```

### parent_id

构建树

### level

快速判断层级、限制深度

### ancestors

快速获取祖先链

- 查询某分类下所有子分类：

```sql
SELECT *
FROM product_category
WHERE category_path LIKE '1/%';
```

- 查询三级分类的完整路径：

```sql
SELECT category_path
FROM product_category
WHERE id = 3;
#1/2/3
```



## product

商品表

```sql
CREATE TABLE product (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '商品ID',
    category_id BIGINT NOT NULL COMMENT '分类ID',

    product_name VARCHAR(255) NOT NULL COMMENT '商品名称',
    product_desc TEXT COMMENT '商品描述',

    cover_image VARCHAR(500) COMMENT '主图',

    price DECIMAL(10,2) NOT NULL COMMENT '销售价',
    original_price DECIMAL(10,2) COMMENT '原价',

    stock INT DEFAULT 0 COMMENT '库存',

    status TINYINT DEFAULT 1 COMMENT '状态(1-上架 0-下架)',

    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_category_id(category_id)
);
```

## cart

购物车表

```sql
CREATE TABLE cart (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    user_id BIGINT NOT NULL COMMENT '用户ID',

    product_id BIGINT NOT NULL COMMENT '商品ID',

    quantity INT NOT NULL COMMENT '购买数量',

    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,

    UNIQUE KEY uk_user_product(user_id, product_id)
);
```

## order

```sql
CREATE TABLE `order` (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    order_no VARCHAR(32) NOT NULL COMMENT '订单号',

    user_id BIGINT NOT NULL COMMENT '用户ID',

    total_amount DECIMAL(10,2) NOT NULL COMMENT '订单总金额',

    pay_amount DECIMAL(10,2) NOT NULL COMMENT '实付金额',

    status TINYINT NOT NULL COMMENT '
        0-待支付
        1-已支付
        2-已发货
        3-已完成
        4-已取消
    ',

    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    pay_time DATETIME NULL,
    finish_time DATETIME NULL,

    UNIQUE KEY uk_order_no(order_no),
    INDEX idx_user_id(user_id)
);
```

根据实际购物场景，如果存在快递场景，可以考虑添加快照字段：收货人手机号、收货人姓名、收货人地址、运费。



## order_item

一个订单对应多个商品。

```sql
CREATE TABLE order_item (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    order_id BIGINT NOT NULL COMMENT '订单ID',

    product_id BIGINT NOT NULL COMMENT '商品ID',

    product_name VARCHAR(255) NOT NULL COMMENT '商品快照名称',

    product_image VARCHAR(500) COMMENT '商品快照图片',

    price DECIMAL(10,2) NOT NULL COMMENT '下单时单价',

    quantity INT NOT NULL COMMENT '购买数量',

    amount DECIMAL(10,2) NOT NULL COMMENT '小计',

    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_order_id(order_id)
);
```

这里的：`product_name`，`product_image`，`price`必须保存。因为这三个字段可能随时会变化：

```
用户下单时：
iPhone 16 = 5999元

一个月后：
iPhone 16 = 6999元
```

订单商品不能实时查询商品表

## order_pay

不建议在order表中维护支付方式，因为用户退款时，需要支付平台的流水号。

```sql
CREATE TABLE order_pay
(
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '支付记录ID',

    order_id BIGINT NOT NULL COMMENT '订单ID',

    pay_no VARCHAR(64) NOT NULL COMMENT '第三方交易号',

    pay_type TINYINT NOT NULL COMMENT '
        支付方式：
        1-支付宝
        2-微信支付
        3-银行卡
        4-余额支付
    ',

    pay_amount DECIMAL(10,2) NOT NULL COMMENT '支付金额',

    pay_status TINYINT NOT NULL COMMENT '
        0-待支付
        1-支付成功
        2-支付失败
        3-已退款
    ',

    pay_time DATETIME NULL COMMENT '支付时间',

    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
);
```



一个订单可能有多条支付记录：

场景1：支付失败重试

| order_id | pay_no | pay_status |
| -------- | ------ | ---------- |
| 1001     | wx001  | 失败       |
| 1001     | wx002  | 成功       |

场景2：支付失败更换支付方式

| order_id | pay_type | pay_no | pay_status |
| -------- | -------- | ------ | ---------- |
| 1001     | 微信     | wx001  | 失败       |
| 1001     | 支付宝   | zfb002 | 成功       |

场景3：余额+微信组合支付

常见于收银台场景下

| order_id | pay_type | pay_amount |
| -------- | -------- | ---------- |
| 1001     | 余额     | 20         |
| 1001     | 微信     | 80         |
