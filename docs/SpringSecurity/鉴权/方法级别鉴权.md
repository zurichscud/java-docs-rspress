# 方法级别鉴权

对于粒度更细、逻辑更复杂的权限，可以通过注解直接加在 Controller 或 Service 方法上：

## 开启功能

在配置类上加 `@EnableMethodSecurity`。

## 声明权限

`@PreAuthorize` 支持使用 **SpEL**。在方法执行前，Spring Security 会解析这个表达式，如果计算结果为 `true` 则放行，为 `false` 则抛出 `AccessDeniedException` (403 异常)。

```java
@RestController
@RequestMapping("/system/user")
public class UserController {

    // 访问此方法需要具备 'sys:user:export' 权限
    @PreAuthorize("hasAuthority('sys:user:export')")
    @GetMapping("/export")
    public Result exportUsers() {
        return Result.success(userService.export());
    }

    // 允许 ADMIN 角色，或者传入的 userId 与当前登录用户 ID 一致
    @PreAuthorize("hasRole('ADMIN') or #userId == authentication.principal.id")
    @GetMapping("/{userId}")
    public Result getUserDetail(@PathVariable Long userId) {
        return Result.success(userService.getById(userId));
    }
}
```

Spring Security 提供了多个内置的 SpEL 函数，可以直接调取当前登录用户的权限和角色：

```java
// 1. 单个权限校验：要求用户拥有某个具体的权限码
@PreAuthorize("hasAuthority('sys:user:add')")
public void addUser() {}

// 2. 多个权限校验（满足其一即可 / 必须同时满足）
@PreAuthorize("hasAnyAuthority('sys:user:add', 'sys:user:edit')") // OR 逻辑
@PreAuthorize("hasAuthority('sys:user:add') and hasAuthority('sys:user:edit')") // AND 逻辑

// 3. 角色校验（Spring Security 会自动在角色名称前拼接 "ROLE_"）
@PreAuthorize("hasRole('ADMIN')") // 匹配 GrantedAuthority 中的 "ROLE_ADMIN"
@PreAuthorize("hasAnyRole('ADMIN', 'MANAGER')")
```



## 原理

方法级别鉴权底层使用的是 **Spring AOP** 切面，在目标方法执行前（`@PreAuthorize`）或执行后（`@PostAuthorize`）调用 `AuthorizationManager` 进行校验。
