# RBAC

## 基本思想

**RBAC**（Role-Based Access Control，基于角色的访问控制）是目前企业级应用中最主流的权限管理模型。

它的核心思想非常朴素：**解耦用户与权限**。不直接给用户分配具体操作权限，而是先将权限赋予“角色”，再把“角色”赋予“用户”。

## 表数据库设计

| **表名**                          | **作用**        | **核心字段示例**                                             |
| --------------------------------- | --------------- | ------------------------------------------------------------ |
| **`sys_user`**                    | 用户表          | `id`, `username`, `password`, `status`                       |
| **`sys_role`**                    | 角色表          | `id`, `role_name`, `role_code` (如 `admin`, `editor`)        |
| **`sys_menu` / `sys_permission`** | 权限/菜单表     | `id`, `parent_id`, `perm_code` (如 `user:add`), `type` (菜单/按钮) |
| **`sys_user_role`**               | 用户-角色关联表 | `user_id`, `role_id`                                         |
| **`sys_role_menu`**               | 角色-权限关联表 | `role_id`, `menu_id`                                         |

### User与 Role的关系

多对多

- **一个用户可以拥有多个角色：**

   例如，员工“张三”既可以是“项目经理”，也可以兼任“高级开发工程师”。他在系统里同时具备这两个角色的所有权限。

- **一个角色可以分配给多个用户：**

   例如，“前端开发”这个角色，可以同时授予“张三”、“李四”、“王五”多名员工。

### Role与 Menu/Permission的关系

多对多

- **一个角色可以拥有多个菜单/权限：**

  例如，“财务专员”角色可以访问 `财务报表`、`出纳管理`、`发票开具` 等多个菜单。

- **一个菜单/权限可以分配给多个角色：**
	
	例如，`系统首页` 或 `个人中心` 菜单可以分配给“管理员”、“普通员工”、“财务专员”等所有角色。
