# 配置类

## 方式一：WebSecurityConfigurerAdapter

**基类继承**：必须继承 `WebSecurityConfigurerAdapter` 并重写 `configure(HttpSecurity http)` 方法。

**方法注解**：开启方法级权限控制需要使用 `@EnableGlobalMethodSecurity(prePostEnabled = true)`。

```java
package com.example.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.method.configuration.EnableGlobalMethodSecurity;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;
import org.springframework.security.config.http.SessionCreationPolicy;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;

@Configuration
@EnableWebSecurity
@EnableGlobalMethodSecurity(prePostEnabled = true) // 2.5 版本开启方法级权限的注解
public class SecurityConfig extends WebSecurityConfigurerAdapter { // 继承基类

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            // 1. 直接使用 .csrf().disable()
            .csrf().disable()
            
            // 2. 支持跨域（会默认去容器中找 CorsConfigurationSource 类型的 Bean）
            .cors()
            
            .and()
            
            // 3. 配置请求拦截
            .authorizeRequests()
          // 只允许匿名用户访问，已经登录的用户不能访问。
            .antMatchers("/api/auth/login", "/api/auth/register").anonymous()
          // 无论登没登录，所有人都能访问这些静态资源和注册接口
            .antMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll()
          // 其他所有请求必须登录
            .anyRequest().authenticated()
            
            .and()
            
            // 4. 前后端分离禁用 Session
            .sessionManagement()
            .sessionCreationPolicy(SessionCreationPolicy.STATELESS);
            
        // 如果有自定义 JWT 过滤器，在这里插入
        // http.addFilterBefore(jwtAuthenticationTokenFilter, UsernamePasswordAuthenticationFilter.class);
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

**`.permitAll()`（推荐用于公共接口）**

- **含义**：所有人都可以访问，**无论登录还是没登录**。
- **场景**：注册接口、静态资源、获取系统公共配置等。已经登录的用户也可以正常调用。

**`.anonymous()`（仅限未登录用户）**

- **含义**：**只有“未登录”的匿名用户可以访问**。一旦用户登录成功拿到了 Token/Session，再次访问该接口就会被拒绝（报 403 错误）。
- **场景**：登录接口（`/login`）、忘记密码找回接口。因为已经登录的用户不需要再访问登录接口。

## 方式二：SecurityFilterChain（推荐）

在 Spring Security 中，现代的配置方式已经全面废弃了原有的 `WebSecurityConfigurerAdapter` 基类，改为**基于组件（Component-based）的配置方式**。

核心思想是通过注册一个类型为 `SecurityFilterChain` 的 `@Bean` 来定义安全过滤链。

```java
package com.example.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.method.configuration.EnableMethodSecurity;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.configurers.AbstractHttpConfigurer;
import org.springframework.security.config.http.SessionCreationPolicy;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.security.web.SecurityFilterChain;
import org.springframework.web.cors.CorsConfiguration;
import org.springframework.web.cors.CorsConfigurationSource;
import org.springframework.web.cors.UrlBasedCorsConfigurationSource;

import java.util.Collections;

@Configuration
@EnableWebSecurity // 开启 Web 安全设置
@EnableMethodSecurity // 开启方法级别的权限注解（如 @PreAuthorize）
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            // 1. 配置跨域与禁用 CSRF（前后端分离通常关闭 CSRF）
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            .csrf(AbstractHttpConfigurer::disable)
            
            // 2. 配置请求拦截规则（Lambda 链式调用）
            .authorizeHttpRequests(auth -> auth
                // 允许匿名访问的静态资源或登录注册接口
                .requestMatchers("/api/auth/login", "/api/auth/register").permitAll()
                .requestMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll()
                // 其他所有请求都需要认证
                .anyRequest().authenticated()
            )
            
            // 3. 调整为无状态 Session（适用于 JWT 架构）
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            );
            
            // 4. 如果有自定义的 JWT 过滤器，在此处插入
            // .addFilterBefore(jwtAuthenticationTokenFilter, UsernamePasswordAuthenticationFilter.class)
            
            // 5. 配置异常处理器（如匿名访问被拒绝、无权限访问）
            // .exceptionHandling(exception -> exception
            //     .authenticationEntryPoint(customAuthenticationEntryPoint)
            //     .accessDeniedHandler(customAccessDeniedHandler)
            // );

        return http.build();
    }

    /**
     * 密码加密组件
     */
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    /**
     * 跨域配置源
     */
    private CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        configuration.setAllowedOriginPatterns(Collections.singletonList("*")); // 允许所有源
        configuration.setAllowedMethods(Collections.singletonList("*")); // 允许所有方法 (GET, POST等)
        configuration.setAllowedHeaders(Collections.singletonList("*")); // 允许所有请求头
        configuration.setAllowCredentials(true); // 允许携带 Cookie
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", configuration);
        return source;
    }
}
```
