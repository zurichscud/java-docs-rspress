# 消息网关

## 会话持久化

会话会在多轮消息之间持续保留，Agent 能够记住当前对话上下文，直到会话被重置。

会话重置策略，支持以下几种重置方式：

| 策略     | 默认值              | 说明                           |
| -------- | ------------------- | ------------------------------ |
| 每日重置 | 凌晨 4:00           | 每天在指定时间自动重置会话     |
| 空闲重置 | 1440 分钟（24小时） | 会话空闲超过指定时间后自动重置 |
| 组合模式 | 两者同时生效        | 以最先触发的条件为准           |

可在 `~/.hermes/gateway.json` 中为不同平台单独配置重置策略：

```json
{
  "reset_by_platform": {
    "telegram": {
      "mode": "idle",
      "idle_minutes": 240
    },
    "discord": {
      "mode": "idle",
      "idle_minutes": 60
    }
  }
}
```

上述配置表示：

- Telegram 会话空闲 240 分钟后重置
- Discord 会话空闲 60 分钟后重置

## 中断Agent

在 agent 工作时发送任意消息即可中断它。关键行为：

- **正在执行的终端命令立即终止**（SIGTERM，1 秒后 SIGKILL）
- **工具调用被取消** — 仅当前正在执行的工具调用会运行，其余跳过
- **多条消息合并** — 中断期间发送的消息合并为一个 prompt
- **`/stop` 命令** — 中断而不排队后续消息

## 后台进程通知

> 当运行后台会话的 agent 使用 `terminal(background=true)` 启动长时间运行的进程（服务器、构建等）时，网关可以向你的聊天推送状态更新。

```
你在微信跟 Hermes 聊天
    ↓
Hermes 派生一个子 agent 执行后台任务（比如编译项目、启动服务器）
    ↓
子 agent 用了 terminal(background=True) 启动了一个耗时进程
    ↓
进程跑完了 / 出错了 / 有重要状态变化
    ↓
Hermes 自动通过网关把结果推送到你的微信
```

假设你对微信里的 Hermes 说：

│ "帮我编译这个项目，完成后通知我"

Hermes 的流程：

1. 派生子 agent
2. 子 agent 执行 terminal(command="npm run build", background=True, notify_on_complete=True)
3. 编译过程中你可以继续聊天
4. 编译完成后，Hermes 通过微信网关发一条消息：✅ 编译完成 或 ❌ 编译失败，错误：...

核心价值

你不需要一直盯着终端等结果。 任务在后台跑，完成后微信/QQ/Telegram 会收到通知，就像手机 app 的推送一样。
