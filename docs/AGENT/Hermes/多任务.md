# 多任务



## 同时处理多个任务

Hermes Agent 提供多种并行处理任务的方式：

###  `delegate_task` （推荐）

在一条消息中描述多个独立任务，Hermes 会自动并行派生子 agent 处理，最后汇总结果。最多支持 **3 个并发子任务**。

示例：

> "帮我同时搜索 A 的文档，和检查 B 项目的测试是否通过"

###  `/background` 命令（CLI 模式）

在 CLI 交互中，用 `/background` 把任务放到后台执行，当前会话可继续聊天：

```
/background 帮我写一个 FastAPI 项目
```

### 终端后台进程

通过 `terminal(background=True)` 启动后台进程，同时处理其他事情。

### tmux 多窗口（完全独立的交互式 Agent）

需要多个完全独立的 Agent 同时工作时，用 tmux 启动多个 hermes 实例：

```bash
tmux new-session -d -s agent1 'hermes'
tmux new-session -d -s agent2 'hermes'
```

### 方式对比

| 场景                  | 推荐方式                 |
| --------------------- | ------------------------ |
| 几个独立的快速子任务  | `delegate_task` 批量模式 |
| 一个长任务 + 继续聊天 | `/background`            |
| 两个长时间独立项目    | tmux 多实例              |

## /agents

`/agents`（别名 `/tasks`）是会话管理命令，用于查看**当前会话内**正在运行的活跃子 agent 和后台任务。

显示内容：

- 通过 `delegate_task` 派生的子 agent 状态
- 通过 `/background` 启动的后台任务
- 通过 `terminal(background=True)` 启动的后台进程

不显示的内容：

- 其他独立的 Hermes 实例（完全隔离，互不可见）

---

`/agents` vs 多个 Hermes 实例：

|            | `/agents`                 | 多个 Hermes 实例         |
| ---------- | ------------------------- | ------------------------ |
| 关系       | 父子关系（同一会话内）    | 陌生人（完全隔离）       |
| 共享上下文 | ✅ 子 agent 结果会汇总回来 | ❌ 各自独立               |
| 互相可见   | ✅ 通过 `/agents` 查看     | ❌ 互不可见               |
| 典型用法   | 并行处理几个快速子任务    | 同时做两个独立的长期项目 |

多个独立 Hermes 实例如需协同工作，可通过以下方式间接协调：

- 文件系统共享：通过本地文档共享信息
- tmux 捕获输出
- Kanban 看板（`hermes kanban`）

## /background

每个 `/background` prompt 会生成一个**独立的 agent 实例**异步运行：

- **隔离会话** — 后台 agent 拥有自己的会话和对话历史。它不了解你当前的聊天上下文，只接收你提供的 prompt。
- **相同配置** — 继承当前网关配置中的模型、提供商、工具集、推理设置和提供商路由。
- **非阻塞** — 你的主聊天保持完全交互。在后台任务运行期间，你可以发送消息、运行其他命令或启动更多后台任务。
- **结果传递** — 任务完成后，结果发送回**发出命令的同一聊天或频道**，前缀为"✅ Background task complete"。如果失败，你会看到"❌ Background task failed"及错误信息。
