# 云服务器

VPS 和 ECS 都可以理解为“云服务器”，但它们的实现方式和资源隔离程度不同。

> **VPS = 一台物理服务器虚拟出来的多个小服务器**
>  **ECS = 云厂商提供的弹性计算服务器（通常也是虚拟机，但底层更复杂、更云化）**

|          | VPS            | ECS                 |
| -------- | -------------- | ------------------- |
| 定义     | 虚拟服务器     | 云服务器实例        |
| 提供商   | VPS厂商较多    | 大型云厂商          |
| 虚拟化   | 常见KVM/OpenVZ | KVM/Xen等企业虚拟化 |
| 弹性扩容 | 一般           | 强                  |
| 快照     | 有些支持       | 标配                |
| 私有网络 | 少             | 标配                |
| 负载均衡 | 少             | 完善                |
| 自动扩容 | 少             | 支持                |
| SLA保障  | 一般           | 较高                |
| 价格     | 便宜           | 较贵                |

## VPS

> Virtual Private Server（虚拟专用服务器）

它通常是在一台物理服务器上，通过虚拟化技术切割出来多个独立环境。

```
物理服务器
│
├── VPS 1（2核4G）
│     └── Linux
│
├── VPS 2（2核4G）
│     └── Linux
│
├── VPS 3（4核8G）
│     └── Linux
```

## 资源超售

VPS 虽然会显示：

```
2核 CPU
4GB 内存
50GB 硬盘
```

但这个“2核”可能只是**虚拟配置**，不一定对应物理服务器上的 2 个真实 CPU 核心。

假设一台物理服务器：

```
VPS A：8核
VPS B：8核
VPS C：8核
...
VPS H：8核
```

VPS 商家可能卖8台VPS。但是有些商家可能卖10台（超过了物理主机的配置），这叫CPU 超售（CPU Overcommit）。原因是商家赌：

不会所有用户同时跑满 CPU。高负载的情况下，就会存在虚拟机竞争资源的情况。

ECS虽然也会超卖，但是会比VPS好点
