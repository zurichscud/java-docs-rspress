# 异常处理

在 Spring MVC 中，**异常处理器（Exception Handler）** 是处理 Controller 层抛出异常的核心机制。它可以让我们告别传统的 `try-catch` 堆砌，实现**集中式、全局性的异常处理**

## 全局异常处理器

这是目前主流开发中最常用的方式。它利用 AOP（面向切面编程）思想，能够拦截所有 Controller 抛出的异常，并直接将对象转换为 JSON 返回给前端。具体参考`@RestControllerAdvice`

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        return Result.fail("系统异常");
    }
}
```



## 局部异常处理器

如果你希望某个异常**只在特定的 Controller 中**被特殊处理，可以直接把 `@ExceptionHandler` 注解写在该 Controller 内部。

- **特点**：它的优先级**高于**全局异常处理器。如果当前 Controller 内部有匹配的处理器，就不会走 `@RestControllerAdvice`。

- **缺点**：无法跨 Controller 复用，代码冗余。

```java
@RestController
@RequestMapping("/user")
public class UserController {

    @GetMapping("/{id}")
    public User getUser(@PathVariable Long id) {
        if (id == 404) {
            throw new UserNotFoundException("用户不存在");
        }
        return new User(id, "张三");
    }

    // 只对当前 UserController 生效
    @ExceptionHandler(UserNotFoundException.class)
    public Map<String, Object> handleUserNotFound(UserNotFoundException e) {
        Map<String, Object> map = new HashMap<>();
        map.put("code", 4041);
        map.put("msg", e.getMessage());
        return map;
    }
}
```



## Spring MVC 异常解析

Spring MVC 的底层实际上是通过 `HandlerExceptionResolver` 接口的实现类来解析异常的。

Spring MVC 内置的常见实现类：

1. **`ExceptionHandlerExceptionResolver`**：负责解析 `@ExceptionHandler` 注解（优先级最高）。
2. **`ResponseStatusExceptionResolver`**：解析带有 `@ResponseStatus` 注解的异常，将其映射为指定的 HTTP 状态码（如 404, 500）。
3. **`DefaultHandlerExceptionResolver`**：Spring 内置的默认兜底解析器，将 Spring 特定的异常（如 `HttpRequestMethodNotSupportedException` 405错误）翻译成对应的 HTTP 状态码。

::: tip

你也可以通过实现 `HandlerExceptionResolver` 接口来自定义异常处理，但它返回的是一个 `ModelAndView` 对象，更适合传统的 JSP 页面跳转，在如今前后端分离（返回 JSON）的趋势下，已经很少使用。

:::
