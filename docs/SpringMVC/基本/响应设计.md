# 响应设计

## 严格遵循 HTTP 规范

Spring Boot 默认

- **做法**：出错时，网络状态码是 400、404、500。业务只以HTTP状态码判断

- **前端处理（Vue）**：Axios 的 `interceptors.response.use()` 会直接进入 **第二个回调函数（`error => {}`）**。前端需要在这个错误拦截器里捕获 `error.response.status` 来做统一提示（如弹窗提示服务器崩溃或跳转 404 页面）。

## 全员 200 派

- **做法**：不管后端是接口报错、参数校验失败、还是数据库挂了，HTTP 网络状态码**永远是 `200 OK`**。真正的错误状态由 JSON 里的自定义字段（如 `code: 50001` 或 `success: false`）来表达。

- **前端处理（Vue）**：Axios 永远会进入 **第一个回调函数（`response => {}`）**。前端通过判断 `res.data.code === 200`（或者 `res.data.success`）来决定是正常走业务逻辑，还是弹窗报错。

::: warning 注意

- **后端绝对无法控住所有的 HTTP 状态码**。基础设施和网络带来的硬性非 200 状态码，前后端依然要有一套兜底方案去和它们对线。
- 因此前端在处理响应时，需要先处理HTTP网络状态码，再处理内部的业务状态码
- 不同公司的成功码可能是200、0、1或是true，并没有唯一标准

:::

## 统一响应设计

如果需要控制 HTTP 状态码，需要使用`ResponseEntity`

```java
@ExceptionHandler(BusinessException.class)
public ResponseEntity<Result<?>> handleBusiness(
        BusinessException e) {

    return ResponseEntity
            .badRequest()
            .body(Result.fail(
                    e.getCode(),
                    e.getMessage()));
}
```
