# 解决重复读取请求体

如果我们在 Filter 中直接通过 `request.getInputStream()` 读取了前端传来的 JSON 参数

后续的 Controller 就无法再次读取该输入流了（流只能读取一次）。

` @RequestBody`会调用`getInputStream`去读取请求体，但是已经无法读取了

**解决方案：** 使用 Spring 提供的 `ContentCachingRequestWrapper` 和 `ContentCachingResponseWrapper` 来包装原始的 request 和 response。

```java
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {

        // 1. 转换为 HttpServletRequest/Response 并用 Wrapper 包装，防止读取 Body 后导致流关闭
        ContentCachingRequestWrapper requestWrapper = new ContentCachingRequestWrapper((HttpServletRequest) request);
        ContentCachingResponseWrapper responseWrapper = new ContentCachingResponseWrapper((HttpServletResponse) response);

        long startTime = System.currentTimeMillis();

        try {
            // 2. 放行请求，让后续的 Controller 执行
            chain.doFilter(requestWrapper, responseWrapper);
        } finally {
            long executeTime = System.currentTimeMillis() - startTime;

            // 3. 开始记录日志（必须在放行后记录，此时包装类才真正抓取到了 Body 数据）
            logRequestDetails(requestWrapper, responseWrapper, executeTime);

            // 4. 关键：将缓存的响应体复制回原始的响应流中，否则前端会接收到空响应
            responseWrapper.copyBodyToResponse();
        }
    }
```
