# 内容协商

在 Spring Boot 中，内容协商（Content Negotiation）是指服务端根据客户端的期望，决定以何种格式（如 JSON、XML、HTML 等）来返回响应数据的机制。

简单来说，就是“同一份数据，多种表达方式”。

## 核心原理

Spring MVC 的内容协商主要依赖于 `ContentNegotiationManager` 和 `HttpMessageConverter`（消息转换器）。其核心流程如下：

1. **客户端发送请求**：通过特定的方式（如请求头、请求参数）告诉服务器它想要什么格式的数据。
2. **服务器匹配媒体类型（Media Type）**：Spring Boot 根据配置和请求信息，确定最终采用的 `MediaType`（如 `application/json`）。
3. **选择转换器**：Spring Boot 遍历注册的 `HttpMessageConverter`，找到能处理该 `MediaType` 且能处理返回对象类型的转换器。
4. **输出数据**：转换器将 Java 对象序列化为对应的格式并写入 HTTP 响应体。

## 客户端触发内容协商的方式

Spring Boot 默认支持以下几种协商方式（按优先级从高到低）：

### 方式一：基于 HTTP 请求头（默认开启）

客户端在请求头中携带 `Accept` 字段，声明自己能接收的类型。

- `Accept: application/json` $\rightarrow$ 返回 JSON
- `Accept: application/xml` $\rightarrow$ 返回 XML

### 方式二：基于请求参数（需手动开启）

通过 URL 中的参数（默认是 `format`）来指定格式。例如：`/user/1?format=xml`。

要在 `application.yml` 中开启：

```yaml
spring:
  mvc:
    contentnegotiation:
      favor-parameter: true # 开启参数协商
      parameter-name: format # 属性名，默认就是 format
```



## 让 Spring Boot 支持 XML 格式

Spring Boot 默认只内置了 Jackson 的 JSON 转换器。如果你访问一个接口并传入 `Accept: application/xml`，通常会收到 `406 Not Acceptable` 错误。

要让 Java 后端支持 XML 输出，非常简单，只需要引入 Jackson 的 XML 兼容包：

```
<dependency>
    <groupId>com.fasterxml.jackson.dataformat</groupId>
    <artifactId>jackson-dataformat-xml</artifactId>
</dependency>
```

引入依赖后，无需修改任何 Java 代码。

- 请求 `Accept: application/json`  返回 `{"name": "Jack"}`
- 请求 `Accept: application/xml `返回 `<User><name>Jack</name></User>`
