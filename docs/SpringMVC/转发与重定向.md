# 转发与重定向
在 Spring MVC（以及传统的 Web 开发）中，转发（Forward） 和 重定向（Redirect） 是两种核心的页面/接口跳转方式。

## 核心区别

| **特性**       | **转发 (Forward)**                          | **重定向 (Redirect)**                      |
| -------------- | ------------------------------------------- | ------------------------------------------ |
| **发生位置**   | **服务器内部**完成。                        | **浏览器（客户端）**重新发起请求。         |
| **浏览器 URL** | **不变**（前端不知道后端悄悄换了地方）。    | **改变**（变为重定向后的新地址）。         |
| **请求次数**   | **1 次**请求。                              | **2 次**独立的请求。                       |
| **数据共享**   | `HttpServletRequest` 域中的**数据不丢失**。 | 原请求的数据**会丢失**（因为是全新请求）。 |
| **跳转范围**   | 只能跳转到**当前应用内部**的资源。          | 可以跳转到**任意外部网站**（如百度）。     |
| **常见状态码** | 200 OK（对前端而言只有一次成功响应）。      | 302 Found 或 303 See Other。               |



## Forward

### 具体流程

1. 浏览器向 `A` 发起请求。
2. 服务器内部的 `A` 组件发现搞不定，或者需要协同，直接在**服务器内部**把请求转交给 `B`。
3. `B` 处理完后，将结果返回给浏览器。 *（整个过程浏览器只发了一次请求，URL 依然显示 `A`）*

### 代码实现

通过原生 `HttpServletRequest` 的 `getRequestDispatcher` 进行转发

```java
@RestController
@RequestMapping("/api/v1/order")
public class OrderController {

    @PostMapping("/submit")
    public void submitOrder(HttpServletRequest request, HttpServletResponse response) throws Exception {
        // 1. 模拟处理前后端分离中常见的上下文数据（如从 Token 中解析出的 userId）
        request.setAttribute("currentUserId", 9527L);
        
        // 2. 执行内部转发
        // 注意：这里的路径必须是完整的后端路由
        request.getRequestDispatcher("/api/v1/payment/pay").forward(request, response);
    }
}

@RestController
@RequestMapping("/api/v1/payment")
public class PaymentController {

    @PostMapping("/pay")
    public ResponseEntity<Map<String, Object>> pay(HttpServletRequest request) {
        // 3. 目标接口无缝获取上一步塞入的参数
        Long userId = (Long) request.getAttribute("currentUserId");
        
        Map<String, Object> result = new HashMap<>();
        result.put("code", 200);
        result.put("msg", "订单已转发并成功拉起支付");
        result.put("userId", userId);
        
        // 4. 返回标准 ResponseEntity 给 Vue 前端
        return ResponseEntity.ok(result);
    }
}
```



## Redirect

### 具体流程

1. 浏览器向 `A` 发起请求。
2. `A` 发现资源搬家了或者流程需要，给浏览器返回一个 **302 状态码**，并在响应头的 `Location` 中写上 `B` 的地址。
3. 浏览器收到后，**自动**向新地址 `B` 发起第二次全新请求。
4. `B` 处理并返回给浏览器。 *（整个过程发了两次请求，浏览器 URL 变成了 `B`）*



### 代码实现

在Spring中，如果需要重定向，通常使用 `HttpServletResponse` 或 `ResponseEntity`。

```java
@RestController
@RequestMapping("/api")
public class ApiController {

    // 前后端分离中的重定向
    @GetMapping("/external-login")
    public void redirectToOauth(HttpServletResponse response) throws IOException {
        // 比如跳转到第三方单点登录 (SSO) 页面
        response.sendRedirect("https://oauth.provider.com/auth");
    }

    // 或者使用 ResponseEntity 优雅构建
    @GetMapping("/old-path")
    public ResponseEntity<Void> oldPath() {
        return ResponseEntity.status(HttpStatus.FOUND) // 302
                .header(HttpHeaders.LOCATION, "/api/v2/new-path")
                .build();
    }
}
```
