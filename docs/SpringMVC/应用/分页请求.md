# 分页请求

## 分页参数合理化

在分页插件 **MyBatis PageHelper** 中，`reasonable` 是一个非常经典且实用的参数，被称为“分页合理化”（Page Rationalization）。

它的核心作用是：**防止用户输入错误的页码（比如负数，或者超过最大页码的数）导致查询报错或返回空数据。**

**场景 A（超出上限）：** 用户访问了第 `6` 页（`pageNum=6`）。

- `reasonable=false`（默认）：MyBatis 会老老实实去执行 `LIMIT 50, 10`，结果返回**空列表**（`[]`）。
- `reasonable=true`（开启）：PageHelper 发现 6 超过了最大页数 5，会自动把页码**修正为最后一页（第 5 页）**，并返回第 5 页的数据。

**场景 B（超出下限）：** 用户传入了第 `0` 页或 `-1` 页（`pageNum=-1`）。

- `reasonable=false`：可能会导致 SQL 报错（因为 `LIMIT` 不能为负数）。
- `reasonable=true`：PageHelper 会自动把页码**修正为第一页（第 1 页）**。
