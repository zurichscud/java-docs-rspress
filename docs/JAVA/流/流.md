# 流

## 流为什么只能读取一次

**流（Stream）之所以只能读取（消费）一次**，核心原因在于它的**设计理念、内存优化**以及**底层物理特征**。

流的本质是**数据的传输通道**，而不是数据的**存储容器**。

- **指针的单向移动**：在读取流时，底层通常有一个指针（Pointer）或游标，每读取一个字节，指针就向前移动一步。
- **不记录历史**：为了效率，流在设计上**不会在内存中缓存已经读过的数据**。数据一旦被你“消费”了，它在流中就消失了。

这就好比你在河边看流水，某一秒流过你眼前的这滴水，一旦流过去，就再也不会倒流回来。如果你想再看一遍这滴水，除非你提前用桶把它装起来（这叫内存缓存）。

## 核心优势：对内存极其友好

为什么不把流设计成可以反复读取的呢？因为**代价太大了**。

试想一下，如果你需要处理一个 **10G** 的大视频文件：

- **如果是用容器（如数组）**：你必须把 10G 的数据全部一次性加载到内存中。如果你的机器只有 8G 内存，程序直接 OOM（内存溢出）崩溃。
- **如果是用流（Stream）**：每次只加载几 KB 的数据到内存，处理完这几 KB，释放掉，再读接下来的几 KB。整个过程中，内存占用极低且恒定（比如一直维持在几兆字节）。

如果流支持重复读取，就意味着它必须在内存中把读过的数据全部存下来，这就违背了流“用最小的内存处理海量数据”的初衷。



```java
// 1. 读取流，把数据存入变量。此时流已经空了！
String result = bufferReader.readLine(); 

System.out.println(result); // 第一次打印：能看到
System.out.println(result); // 第二次打印：还能看到: 因为你打印的是 result 这个内存变量,而不是流本身
```



```java
// 第一次读取并打印
System.out.println(bufferReader.readLine()); // 能看到内容

// 第二次尝试读取并打印
System.out.println(bufferReader.readLine()); // 输出 null 或者报错！
```
