# 概述

Java 本质上是一门面向对象（OOP）语言，因此它实现函数式编程的方式是**以面向对象为底层支撑，用语法糖和接口封装来提供函数式体验**。



Java 中没有独立的“函数”类型，函数必须依托于接口存在。**只要一个接口有且仅有一个抽象方法**，它就是函数式接口（通常带有 `@FunctionalInterface` 注解）。

| **接口类型**         | **抽象方法 signature** | **入参** | **返回值**  | **典型应用场景 / 示例**                  |
| -------------------- | ---------------------- | -------- | ----------- | ---------------------------------------- |
| **`Function<T, R>`** | `R apply(T t)`         | 有 (`T`) | 有 (`R`)    | 类型转换、映射：`String::length`         |
| **`Consumer<T>`**    | `void accept(T t)`     | 有 (`T`) | 无 (`void`) | 消费数据、修改对象（如 `Customizer<T>`） |
| **`Supplier<T>`**    | `T get()`              | 无       | 有 (`T`)    | 延迟加载、对象创建：`ArrayList::new`     |
| **`Predicate<T>`**   | `boolean test(T t)`    | 有 (`T`) | `boolean`   | 条件判断、过滤：`s -> s.isEmpty()`       |
