# 请求



## 请求头分类

请求头（Request Headers）主要是给服务器（Server）看的。

### 身份验证

Authorization

例如：`Authorization: Bearer <token>`，后端会读取这个头，解析里面的 JWT Token，判断当前用户是谁、有没有权限访问这个接口。

### 请求体描述

Content-Type、Content-Length

例如：`Content-Type: application/json`，告诉 Java 后端：我的 Body 里面装的是 JSON 数据”

### 内容协商

告诉服务器希望返回什么内容给客户端

Accept、Accept-Language、Accept-Encoding

例如：`Accept: application/json`，告诉服务器：“我前端只要 JSON 格式的返回值，别给我返回一堆 HTML 页面。”

### CORS

表明请求来源

Host、Origin、Referer

- Origin

`Origin` 的核心作用是**标识请求的真实来源**。

Origin只有在CORS时才会添加到请求头中

`Origin` 只包含请求发起的**协议、域名和端口号**。它**不包含**任何路径（Path）或查询参数。

```
https://juejin.cn
```

- Referer

`Referer` 用来表示：当前请求是从哪个页面跳转或发起的。

只要是浏览器发起的请求，`Referer`都会添加到请求头中

`Referer`包含完整的 **协议、域名、端口、路径以及查询参数**。

```
https://juejin.cn/post/123456789?from=sidebar
```



## 禁止修改的请求头

W3C 和浏览器规范将 `Origin`、`Referer`、`Cookie`、`Host` 等对安全至关重要的请求头列为了 **“被禁止修改的请求头”（Forbidden Header Name）**。这些头只能由浏览器自动控制，JS 没有任何修改权限。

```js
// 浏览器会直接报错，或者忽略这行代码
fetch('http://api.example.com/data', {
  headers: {
    'Origin': 'http://hacker.com' 
  }
});
```
