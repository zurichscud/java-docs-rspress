# 事件驱动开发

**事件驱动开发（Event-Driven Development）** 是一种非常优雅的解耦机制。它允许组件之间在不直接相互引用的情况下进行通信，完美契合了“高内聚、低耦合”的系统设计原则。

Spring Boot 内置了对事件驱动的完美支持，核心主要由三部分组成：**事件（Event）**、**发布者（Publisher）** 和 **监听器（Listener）**。

## 场景

| **行业/场景**          | **典型事件链（Event Chain）**                                |
| ---------------------- | ------------------------------------------------------------ |
| **电商系统（最典型）** | 用户下单成功（发布事件） ➔ 减库存、通知物流、发放积分、触发 30 分钟未支付自动取消定时器。 |
| **社交/内容平台**      | 用户发布了一条视频（发布事件） ➔ 审核系统切片审核、通知粉丝推送、大数据给该用户打标签。 |
| **SaaS/企业管理系统**  | 员工离职（发布事件） ➔ 自动注销企业邮箱、HR 归档、财务结算、IT 资产回收流程启动。 |



## 核心组件

1. **事件（Event）：** 承载数据的载体。它可以是任意的普通 Java 对象。
2. **事件发布者（Event Publisher）：** 负责把事件广播出去。Spring 提供了 `ApplicationEventPublisher` 接口。
3. **事件监听器（Event Listener）：** 负责捕捉并处理事件。使用 `@EventListener` 注解即可轻松实现。



## 代码示例

假设我们有一个用户注册的场景：用户注册成功后，系统需要**发送欢迎邮件**并**发放优惠券**。

### 定义事件

```java
public class UserRegisteredEvent {
    private final String username;
    private final String email;

    public UserRegisteredEvent(String username, String email) {
        this.username = username;
        this.email = email;
    }

    // Getters
    public String getUsername() { return username; }
    public String getEmail() { return email; }
}
```

### 发布事件

在用户服务（UserService）中注入 `ApplicationEventPublisher` 并发布事件：

```java
@Service
public class UserService {

    @Autowired
    private ApplicationEventPublisher eventPublisher;

    public void registerUser(String username, String email) {
        // 1. 执行注册逻辑（保存到数据库等）
        System.out.println("用户注册成功: " + username);

        // 2. 发布事件
        UserRegisteredEvent event = new UserRegisteredEvent(username, email);
        eventPublisher.publishEvent(event);
    }
}
```

### 监听并处理事件

我们可以定义多个监听器，它们会同时收到这个事件并各自处理，彼此互不干扰：

```java
@Component
public class EmailListener {

    @EventListener
    public void sendWelcomeEmail(UserRegisteredEvent event) {
        System.out.println("发送欢迎邮件至: " + event.getEmail());
    }
}
```

```java
@Component
public class CouponListener {

    @EventListener
    public void grantNewUserCoupon(UserRegisteredEvent event) {
        System.out.println("为用户 " + event.getUsername() + " 发放新人优惠券");
    }
}
```

## 异步事件

**默认情况下，Spring 的事件是同步的。** 也就是说，发布事件的方法会等待所有监听器执行完毕后才继续向下执行（在同一个线程内）。 如果发送邮件很慢，会阻塞注册流程。我们可以开启异步执行：

1. 在启动类或配置类上添加 `@EnableAsync`。
2. 在监听器方法上添加 `@Async` 注解。

```java
@Async
@EventListener
public void sendWelcomeEmail(UserRegisteredEvent event) {
    // 此时该方法会在独立的线程池中异步执行
    System.out.println("异步发送邮件中...");
}
```

## 事务绑定事件

在数据库操作中，我们通常希望**只有当事务成功提交之后**，才触发后续操作（比如发货、发短信）。如果事务回滚了，事件就不应该执行。

Spring 提供了 `@TransactionalEventListener` 来解决这个问题：

```java
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
public void handleOrderPaid(OrderPaidEvent event) {
    // 只有在当前事务成功 commit 之后，才会执行该监听逻辑
    System.out.println("订单付款成功，准备发货...");
}
```

## 监听器的执行顺序

如果是同步事件，可以通过 `@Order` 注解来控制多个监听器的消费顺序，数值越小优先级越高：

```java
@EventListener
@Order(1) // 最先执行
public void firstHandler(UserRegisteredEvent event) { ... }
```
