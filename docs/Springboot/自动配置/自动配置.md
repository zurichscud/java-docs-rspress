# 自动配置原理

## spring-boot-autoconfigure

- 每个场景启动器都引入了一个`spring-boot-starter`

- 核心场景启动器中引入了`spring-boot-autoconfigure`包

- `spring-boot-autoconfigure`中囊括了所有场景的所有配置

- 只要这个包下的所有类都能生效，那么相当于Springboot官方写好的整合功能就生效

- **Springboot默认扫描不到`spring-boot-autoconfigure`中写好的所有配置类**，因此需要`@EnableAutoConfiguration`

## @EnableAutoConfiguration

`@EnableAutoConfiguration`是`@SpringbootApplication`的子注解

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@AutoConfigurationPackage
@Import(AutoConfigurationImportSelector.class)
public @interface EnableAutoConfiguration {

    String ENABLED_OVERRIDE_PROPERTY = "spring.boot.enableautoconfiguration";
}
```

当你标注了 `@EnableAutoConfiguration`（或者用 `@SpringBootApplication`）时，Spring 会触发一个关键组件：`AutoConfigurationImportSelector`。

`AutoConfigurationImportSelector`的工作流程如下：

1. **寻找候选名单**：它会去扫描所有 Jar 包下的 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 文件（老版本是 `spring.factories`）。
2. **加载配置类**：这些文件里记录了几百个全限定类名，比如 `RabbitAutoConfiguration`、`DataSourceAutoConfiguration` 等。
3. **条件过滤**：这是最关键的一步。它不会把这几百个配置全部加载，而是根据 `@ConditionalOnClass` 等注解判断：**“你的类路径下有 MySQL 驱动吗？有的话我才加载 DataSource 配置。”**

## XXXAutoConfiguration

- 自动配置类用于向容器中注册组件

- 每个自动配置类都可能存在`@EnableConfigurationProperties`用于将配置文件中指定属性值绑定到指定配置类中

```java
@EnableConfigurationProperties(ServerProperties.class)
```
