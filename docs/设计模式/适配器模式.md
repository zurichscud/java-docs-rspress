# 适配器模式

如果用一句话来概括它的核心思想，那就是：**“变接口，不变逻辑”**。它的作用就像我们日常生活中用的 **电源转换头** 或 **Type-C 转接线** —— 在不修改现有代码的前提下，让原本因为接口不兼容而无法一起工作的类能够协同工作。

## 痛点：第三方 SDK 接口乱七八糟

**接入第三方 SDK**：你接入了两个不同的支付 SDK（微信支付、支付宝），它们的底层 SDK 方法名完全不一样，你想在自己的业务代码里用统一的接口去调用它们。

假设你的商城系统定义了一个统一的支付标准接口 `PaymentService`：

```java
// 商城系统内部定义的统一支付接口
public interface PaymentService {
    // 统一要求：传入订单号和金额（分）
    boolean pay(String orderId, long amountInCents);
}
```

但是，你引入的两个第三方 SDK 方法名和参数完全不一致：

```java
// 微信支付 SDK（第三方提供，无法修改其源码）
public class WechatPaySdk {
    public boolean doWechatPay(String outTradeNo, int totalFee) {
        System.out.println("调用微信 SDK: 订单 " + outTradeNo + "，扣款 " + totalFee + " 分");
        return true;
    }
}

// 支付宝 SDK（第三方提供，无法修改其源码）
public class AlipaySdk {
    // 支付宝参数单位是“元”，且参数名叫 tradeNo
    public void executeAliPay(String tradeNo, double amountInYuan) {
        System.out.println("调用支付宝 SDK: 订单 " + tradeNo + "，扣款 " + amountInYuan + " 元");
    }
}
```

如果直接在业务代码里 `if-else` 判断，业务层就会和具体的第三方 SDK 强绑定，以后新增云闪付、PayPal 时，业务代码会被改得乱七八糟。

## 支付适配器

我们为微信支付和支付宝分别编写一个适配器，实现系统统一的 `PaymentService` 接口，在适配器内部做**参数转换**和**方法重定向**。

- 微信支付适配器

```java
public class WechatPayAdapter implements PaymentService {

    private final WechatPaySdk wechatPaySdk;

    public WechatPayAdapter(WechatPaySdk wechatPaySdk) {
        this.wechatPaySdk = wechatPaySdk;
    }

    @Override
    public boolean pay(String orderId, long amountInCents) {
        // 适配逻辑：把系统的 pay(...) 转换为微信的 doWechatPay(...)
        return wechatPaySdk.doWechatPay(orderId, (int) amountInCents);
    }
}
```

- 支付宝适配器

```java
public class AlipayAdapter implements PaymentService {

    private final AlipaySdk alipaySdk;

    public AlipayAdapter(AlipaySdk alipaySdk) {
        this.alipaySdk = alipaySdk;
    }

    @Override
    public boolean pay(String orderId, long amountInCents) {
        // 适配逻辑 1：单位转换（分 -> 元）
        double amountInYuan = amountInCents / 100.0;
        
        // 适配逻辑 2：调用支付宝 SDK
        alipaySdk.executeAliPay(orderId, amountInYuan);
        
        return true;
      }
}
```

## Service层统一调用

有了适配器后，业务层的下单逻辑对底层的 SDK 差异**完全感知不到**：

```java
public class OrderService {

    // 依赖注入统一的支付接口，不需要关心具体是微信还是支付宝
    private final PaymentService paymentService;

    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }

    public void checkout(String orderId, long amountInCents) {
        System.out.println("=== 正在处理订单支付 ===");
        boolean success = paymentService.pay(orderId, amountInCents);
        if (success) {
            System.out.println("订单支付成功！");
        }
    }
}
```



## PaymentFactory

```java
@Component("WECHAT") // Bean 名称与 PayType 枚举对应
public class WechatPayAdapter implements PaymentService {
    @Override
    public boolean pay(String orderId, long amountInCents) {
        System.out.println("【微信适配器】调用微信 SDK 下单: " + orderId);
        return true;
    }
}

@Component("ALIPAY") // Bean 名称与 PayType 枚举对应
public class AlipayAdapter implements PaymentService {
    @Override
    public boolean pay(String orderId, long amountInCents) {
        System.out.println("【支付宝适配器】转换金额为元并调用支付宝 SDK: " + orderId);
        return true;
    }
}
```

```java
@Component
public class PaymentFactory {

    // Spring 会自动把所有实现了 PaymentService 的 Bean 注入到这个 Map 里
    // Key 为 Bean 的名字（"WECHAT" / "ALIPAY"），Value 为对应的适配器对象
    private final Map<String, PaymentService> payStrategyMap;

    public PaymentFactory(Map<String, PaymentService> payStrategyMap) {
        this.payStrategyMap = payStrategyMap;
    }

    // 根据枚举类型选择对应的适配器
    public PaymentService getPayAdapter(PayType payType) {
        PaymentService adapter = payStrategyMap.get(payType.name());
        if (adapter == null) {
            throw new IllegalArgumentException("暂不支持该支付方式: " + payType);
        }
        return adapter;
    }
}
```



## Controller

Controller 接收前端的 JSON 数据，交由工厂匹配对应的支付适配器并执行：

```java
@RestController
@RequestMapping("/api/orders")
public class OrderController {

    private final PaymentFactory paymentFactory;

    // 构造器注入工厂
    public OrderController(PaymentFactory paymentFactory) {
        this.paymentFactory = paymentFactory;
    }

    @PostMapping("/pay")
    public ResponseEntity<String> payOrder(@RequestBody PayRequest request) {
        // 1. 根据前端传过来的 payType (WECHAT / ALIPAY) 拿到具体的支付适配器
        PaymentService payAdapter = paymentFactory.getPayAdapter(request.getPayType());

        // 2. 统一调用 pay() 方法， Controller 完全不需要知道底层是微信还是支付宝
        boolean success = payAdapter.pay(request.getOrderId(), request.getAmountInCents());

        if (success) {
            return ResponseEntity.ok("支付成功");
        } else {
            return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body("支付失败");
        }
    }
}
```
