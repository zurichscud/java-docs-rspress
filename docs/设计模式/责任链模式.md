

# 责任链模式(洋葱模型)

让数据像流水线一样在关卡里走一遍。

## 什么是责任链

想象一下，你在一家公司上班，今天想**请假 10 天**。 你把请假条交给了你的**小组长**。

1. **小组长**一看：“妈呀，10天？我只有权限批 **1天** 的假。管不了管不了。”（**拒绝处理，一脚踢给下一个人**）
2. 小组长把假条交给了**部门经理**。
3. **部门经理**一看：“10天有点多啊，我最多只能批 **3天** 的假。我也管不了。”（**再次拒绝，继续往前踢**）
4. 部门经理把假条交给了**总经理**。
5. **总经理**一看：“行吧，公司最近不忙，准了！”（**最终解决**）

在这个过程中：

- 你的请假条就是 **“请求”**。
- 小组长 ➔ 部门经理 ➔ 总经理，他们连起来就是一条 **“责任链”**。
- 每个人都在说：“**能处理我直接处理，不能处理我丢给下一个人**”。

## 责任链的优势

### 无责任链

如果你不用这个模式，写代码就像在**玩迷宫**：

> 如果请假天数小于1天，找小组长； 否则，如果小于3天，找部门经理； 否则，如果小于10天，找总经理； 否则……

这样写的坏处是，如果公司突然**增加了一个“总监”级别**，或者**总经理请假了要换人**，你得把整段复杂的判断逻辑全删了重写，极其容易出错。

```java
public class LeaveApprovalSystem {
    
    // 所有的审批逻辑，全都挤在一起
    public void handleLeaveRequest(String employeeName, int days) {
        
        System.out.println(employeeName + " 发起了 " + days + " 天的请假申请。");
        
        // 核心痛点：所有的 if-else 嵌套，层层硬编码
        if (days <= 1) {
            System.out.println("【小组长】审批通过。");
        } else {
            // 超过1天，进入经理审批逻辑
            if (days <= 3) {
                System.out.println("【部门经理】审批通过。");
            } else {
                // 超过3天，进入总监审批逻辑
                if (days <= 7) {
                    System.out.println("【总监】审批通过。");
                } else {
                    // 超过7天，进入总经理审批逻辑
                    if (days <= 30) {
                        System.out.println("【总经理】审批通过。");
                    } else {
                        System.out.println("【系统拒绝】请假天数过长，不予批准！");
                    }
                }
            }
        }
    }
}
```

如果公司今天突然决定：**“取消总监职位，以后超过3天的假直接由总经理审批”**。 你必须钻进这一大堆 `if-else` 的最深处，小心翼翼地把总监的那一层判定删掉，还要重新调整花括号 `{}` 的配对。一不小心少写一个括号，整个系统直接报错。

### 有责任链

而用了“责任链模式”之后，代码变得像**拼积木**： 你只需要把“组长”、“经理”、“总经理”像链条一样拼在一起。

```java
public abstract class LeaveHandler {
    protected LeaveHandler nextHandler; // 链条里的下一个守关人

    // 用来把大家串联在一起的方法
    public void setNextHandler(LeaveHandler nextHandler) {
        this.nextHandler = nextHandler;
    }

    // 核心处理逻辑：子类去实现
    public abstract void handleRequest(String name, int days);
}
```

- 小组长（只管 1 天以内的假）

```java
public class LeaderHandler extends LeaveHandler {
    @Override
    public void handleRequest(String name, int days) {
        if (days <= 1) {
            System.out.println("【小组长】审批通过。员工：" + name + "，天数：" + days);
        } else if (nextHandler != null) {
            // 超过1天，组长管不了，直接踢给下一个人
            nextHandler.handleRequest(name, days);
        }
    }
}
```

- 部门经理（只管 3 天以内的假）

```java
public class ManagerHandler extends LeaveHandler {
    @Override
    public void handleRequest(String name, int days) {
        if (days <= 3) {
            System.out.println("【部门经理】审批通过。员工：" + name + "，天数：" + days);
        } else if (nextHandler != null) {
            // 超过3天，经理也管不了，继续往后踢
            nextHandler.handleRequest(name, days);
        }
    }
}
```

-  总经理（管 30 天以内的假，或者负责最后拒绝）

```java
public class GeneralManagerHandler extends LeaveHandler {
    @Override
    public void handleRequest(String name, int days) {
        if (days <= 30) {
            System.out.println("【总经理】审批通过。员工：" + name + "，天数：" + days);
        } else {
            System.out.println("【总经理拒绝】" + name + " 你想请 " + days + " 天假？直接开除！");
        }
    }
}
```

最后，我们只需要在要用到的地方，把这些独立的关卡“咔哒”一声拼成一条链子。

```java
public class Main {
    public static void main(String[] args) {
        // 1. 实例化各个独立关卡
        LeaveHandler leader = new LeaderHandler();
        LeaveHandler manager = new ManagerHandler();
        LeaveHandler generalManager = new GeneralManagerHandler();

        // 2. 自由组装链路（这就是责任链的精髓！）
        // 组长 ➔ 经理 ➔ 总经理
        leader.setNextHandler(manager);
        manager.setNextHandler(generalManager);

        // 3. 提交申请，永远只找第一个人（组长）就行了
        System.out.println("--- 张三请假 1 天 ---");
        leader.handleRequest("张三", 1);

        System.out.println("\n--- 李四请假 3 天 ---");
        leader.handleRequest("李四", 3);

        System.out.println("\n--- 王五请假 50 天 ---");
        leader.handleRequest("王五", 50);
    }
}
```

## 符合客观规律

想象一下你冬天出门：

1. 先穿内衣（A）
2. 再穿毛衣（B）
3. 最后穿大衣（C）

当你想脱衣服的时候呢？你不可能先脱内衣，你必须：

1. 先脱大衣（C）
2. 再脱毛衣（B）
3. 最后脱内衣（A）

这就形成了一个完美的 **A ➔ B ➔ C ➔（干正事）➔ C ➔ B ➔ A** 的闭环。
