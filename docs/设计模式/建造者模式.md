# 建造者模式

建造者模式（Builder Pattern）

它的核心思想是：**将一个复杂对象的“构建过程”与“它的表示”分离，使得同样的构建过程可以创建不同的表示。**

用一句大白话来说：**“如果创建一个对象太复杂、参数太多、步骤有先后，那就把创建过程交给一个专职的‘建造者’来一步步拼装，最后一口气拿成品。”**

## 为什么需要建造者模式

假设你要创建一个 `Computer`（电脑）对象，电脑有很多配件：CPU、内存、硬盘、显卡、显示器、键盘等。

### 方案一：传统构造函数（重载地狱）

如果有些配件是必选的，有些是可选的，你可能需要写很多构造函数：

```java
// 只有一个必需参数
public Computer(String cpu) { ... }
// 加上内存
public Computer(String cpu, String ram) { ... }
// 加上硬盘
public Computer(String cpu, String ram, String hdd) { ... }
// 加上显卡...
public Computer(String cpu, String ram, String hdd, String gpu, String monitor, String keyboard) { ... }
```

**痛点**：构造函数极长，遇到类型相同的参数（比如好几个 `String` 或 `boolean`），你很容易**按错位置**导致 Bug；且后续加新字段时非常难维护。

### 方案二：Set 方法（无状态/不安全）

先 `new Computer()`，然后不断 `setXxx()`：

```java
Computer computer = new Computer();
computer.setCpu("i7");
computer.setRam("32G");
// 忘记 set 硬盘了...
```

**痛点**：对象在创建过程中处于**“半成品”状态**（线程不安全），而且无法做“必填项校验”或创建不可变（Immutable）对象。

## 建造者模式怎么解决

建造者模式把对象的创建交给内部的 `Builder` 慢慢配置，全部配置好后再调用 `build()` 方法统一校验并生成**不可变的最终对象**。

```java
public class Computer {
    // 假设这些字段都是 final，一旦创建不可修改
    private final String cpu;     // 必填
    private final String ram;     // 必填
    private final String hdd;     // 可选
    private final String gpu;     // 可选

    // 私有构造函数，只能通过 Builder 创建
    private Computer(Builder builder) {
        this.cpu = builder.cpu;
        this.ram = builder.ram;
        this.hdd = builder.hdd;
        this.gpu = builder.gpu;
    }

    // 静态内部类 Builder
    public static class Builder {
        private final String cpu; // 必填
        private final String ram; // 必填
        private String hdd;       // 可选
        private String gpu;       // 可选

        // 必填参数放在 Builder 的构造函数里
        public Builder(String cpu, String ram) {
            this.cpu = cpu;
            this.ram = ram;
        }

        // 可选参数通过链式方法设置，返回 Builder 本身
        public Builder setHdd(String hdd) {
            this.hdd = hdd;
            return this; // 返回 this 是实现链式调用的关键
        }

        public Builder setGpu(String gpu) {
            this.gpu = gpu;
            return this;
        }

        // 最终构建入口
        public Computer build() {
            // 在这里可以做统一的合法性校验
            if (this.cpu == null || this.ram == null) {
                throw new IllegalStateException("CPU 和内存为必填项！");
            }
            return new Computer(this);
        }
    }
}
```

使用方式（流畅的链式调用）：

```java
// 像拼积木一样，需要什么拼什么，顺序自由，可读性极高
Computer myPC = new Computer.Builder("i7", "32G")
        .setHdd("1TB NVMe")
        .setGpu("RTX 4080")
        .build();
```

## 函数式编程

```java
http
    .cors(cors -> cors.configurationSource(mySource)) // CORS 的配置局限在这个 lambda 内
    .csrf(csrf -> csrf.disable())                    // CSRF 的配置局限在这个 lambda 内
    .sessionManagement(session -> session             // Session 的配置局限在这个 lambda 内
        .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
    );
```

HttpSecurity是一个Builder

```java
// HttpSecurity 内部的真实逻辑（简化版）
public HttpSecurity csrf(Customizer<CsrfConfigurer<HttpSecurity>> csrfCustomizer) throws Exception {
    // 1. 拿到或创建一个 CsrfConfigurer 对象
    CsrfConfigurer<HttpSecurity> configurer = getOrApply(new CsrfConfigurer<>());
    
    // 2. 执行你传进来的 Lambda 表达式（即：csrf -> csrf.disable()）
    csrfCustomizer.customize(configurer); 
    
    // 3. 返回 HttpSecurity 自己，继续外层的链式调用
    return this; 
}
```

它本质上就是**把配置对象（`CsrfConfigurer`）当成参数传给你的 Lambda 表达式，让你在闭包里对它进行定制，最后自动返回外层的构建器**。
